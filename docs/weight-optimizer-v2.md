# 权重优化引擎 V2 — 排序质量优化

## 为什么重写

### V1 的问题

老优化器 `optimize_weights.py` 的优化目标是信号列表经过阈值过滤后的推送池命中率：

```
信号列表(400只) → 加权总分 → 阈值≥35 → 推送池(≈5只) → 命中率
                                                    ^^^^^^^^
                                               样本量太小，统计不稳定
```

权重调优只看推送池的命中率，3~5 只股票的命中率波动极大，无法反映权重本身的优劣。

### V2 的新思路

优化目标改为**排序质量**——好的权重应该让涨停股在排序中**集中在前列**：

```
信号列表(400只，有5维分 + 是否涨停)
    ↓
枚举权重 → 加权Top3择优 → 全量排序
    ↓
评估：涨停股在排序中的分布质量
```

## 评分函数：加权 Top3 择优

```
对每只股票:
  1. 计算5个维度的加权贡献分 = 维度原始分 × 权重
  2. 按加权贡献分降序排列
  3. 取前3个维度（加权贡献最高的3个）
  4. 总分 = 这3个维度的加权平均

公式:
  dims_sorted = sort(dims, by=raw_score × weight, desc)
  top3 = dims_sorted[:3]  # 加权贡献最高的3维
  total = Σ(raw_i × weight_i for i in top3) / Σ(weight_i for i in top3)
```

### 与无权重 Top3 的区别

| 方式 | 选Top3规则 | 总分计算 | 权重的意义 |
|------|-----------|---------|-----------|
| 无权Top3（现代码） | 按原始分 | 原始分均值 | ❌ 权重不存在 |
| **加权Top3（新）** | **按 原始分×权重** | **加权均值** | ✅ 权重决定哪个维度进Top3 + 贡献大小 |

### 示例

```
四环生物: 基本面90 技术28 资金40 情绪23 短线17
权重: fund=1.5 tech=1.0 ff=0.5 sent=1.2 st=1.5

加权贡献:
  fundamental: 90×1.5 = 135  ← Top3 ✅
  sentiment:   23×1.2 = 27.6 ← Top3 ✅
  technical:   28×1.0 = 28   ← Top3 ✅
  fundflow:    40×0.5 = 20   ← 第4不进
  shortterm:   17×1.5 = 25.5 ← 第5不进

总分 = (90×1.5 + 23×1.2 + 28×1.0) / (1.5+1.2+1.0)
     = (135 + 27.6 + 28) / 3.7
     = 51.5  ✅
```

## 数据来源

### 输入

- `data/analysis/YYYYMMDD_HHMM.json` — 每日扫描分析结果
  - 每只股票包含：`code`, `name`, `scores`(5维分), `weights`
- Tushare `limit_list_d` API — 拉取对应日期实际涨停股票列表

### 数据流

```
analysis/*.json → 解析 scores + weights + code + trade_date
                → Tushare 拉实际涨停 → 标记 is_limit_up
                → 枚举权重组合 → 加权Top3排序 → 计算指标
```

## 评估指标

### 核心指标（多目标）

好的权重应该同时优化以下指标：

| 指标 | 公式 | 说明 | 目标 |
|------|------|------|------|
| **涨停股平均排名** | `avg(rank_of_limit_up_stocks)` | 所有涨停股在排序中的平均位置 | 越小越好 |
| **涨停股中位排名** | `median(rank_of_limit_up_stocks)` | 抗离群值，比平均更鲁棒 | 越小越好 |
| **Top-K 覆盖率** | `limit_ups_in_top_K / total_limit_ups` | Top 20/50/100 包含了多少比例的涨停股 | 越高越好 |
| **排序分差** | `avg_score(limit_ups) - avg_score(non_limit_ups)` | 权重能否拉大涨停/非涨停的分差 | 越大越好 |
| **AUC** | 排序中随机取涨停 vs 非涨停，涨停排前面的概率 | 综合排序质量 | >0.5, 越大越好 |

### 综合评分公式

```
综合分 = w1 × (1 - 涨停均排/总记录数)   # 排名归一化
       + w2 × Top20覆盖率
       + w3 × 分差归一化
       + w4 × (AUC - 0.5) × 2          # 归一化到0~1
```

权重 `w1~w4` 可调，默认 `w1=0.3, w2=0.4, w3=0.15, w4=0.15`。

## 维度贡献追踪（新增）

这是正反馈的核心——跟踪每个维度在涨停股中被选入 Top3 的频次：

```
统计每日85只涨停股中，各维度跻身Top3的次数：

  基本面  Top3出现率: 85/85 (100%)  ← 核心维度
  技术面  Top3出现率: 60/85  (71%)  ← 重要维度
  资金面  Top3出现率: 30/85  (35%)  ← 辅助维度
  情绪面  Top3出现率: 12/85  (14%)  ← 待数据修复
  短线博弈 Top3出现率:  5/85  (6%)  ← 待接口修复
```

### 正反馈闭环

```
子agent数据源修复 → 维度分提升 → Top3出现率上升
                                      ↓
                             优化器能更准确评估权重
                                      ↓
                             推荐更合理的权重组合
                                      ↓
                             推送池覆盖更多涨停股
```

## 阈值校准曲线（新增）

优化器输出**阈值-覆盖率曲线**，辅助选择推送阈值：

```
以加权Top3分数段统计实际涨停股的分布：

  分数段      涨停数   累计覆盖率
  ≥50:         12只     14%     ← 极严
  ≥45:         22只     26%
  ≥40:         35只     41%
  ≥35:         50只     59%     ← 推荐候选
  ≥30:         65只     76%
  ≥25:         75只     88%
  ≥20:         80只     94%
  <20:          5只      6%     ← 子agent识别盲区

推荐阈值: 35-40（覆盖41-59%涨停股，每次推送约15-25只）
```

## 搜索算法

### 搜索空间

```
5维度 × 每个维度 0.2~3.0，步长 0.1
→ 暴力枚举 29^5 ≈ 2000万种
→ 随机采样 + 局部搜索
```

### 策略

1. **随机采样**：生成 5000~10000 组随机权重组合
2. **基准组合**：当前权重、均衡权重、历史最优权重作为基准
3. **局部精化**：从 Top 10 最优结果出发，邻域微调（±0.1）再搜索一轮
4. **输出 Top 10**：含综合分、各维度贡献率、阈值曲线、对比基准提升幅度

### 约束

- 每个维度权重 >= 0.1（不能完全关闭某个维度）
- 不加阈值过滤——所有信号列表成分全部参与排序
- 权重可以自由变化，不强制归一化

## 与现有复盘机制的关系

| 模块 | V1 优化器 | V2 优化器 | 复盘 AB 对比 |
|------|-----------|-----------|-------------|
| 目标 | 推送命中率 | 排序质量 | 新/旧权重命中率对比 |
| 评分函数 | 加权总分 | 加权Top3择优 | 加权Top3择优 |
| 数据 | 分析文件 | 分析文件 | 当日信号列表 |
| 频率 | 按需 | 按需 | 每日 18:00 |
| 样本 | 推送池(~5只) | 全信号列表(~400只) | 推送池(~20只) |

V2 优化器**不替代**复盘 AB 对比——两者互补：
- 复盘 AB：每日监控推送效果，验证新权重是否有效
- V2 优化器：离线搜索全局最优权重，为调参提供方向
