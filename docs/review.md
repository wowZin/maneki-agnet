# 复盘机制

## 概述

每日 15:30 收盘后，zt-team-leader 自动触发复盘流程，对比预测与实际结果，评估各子 agent 表现，并给出权重调整建议。

## 复盘数据来源

| 数据 | 来源 | 路径 |
|---|---|---|
| 当日预测记录 | zt-team-leader | /srv/zt-team-leader/data/predictions/YYYY-MM-DD.json |
| 实际涨跌结果 | 行情API | 盘后数据接口 |
| 历史权重配置 | zt-team-leader | /srv/zt-team-leader/data/weights.json |
| 历史复盘存档 | zt-team-leader | /srv/zt-team-leader/data/reviews/YYYY-MM-DD.json |

## 复盘流程

### 1. 结果匹配

```text
对当日每条预测:
  - 查询该股票当日最终涨跌幅
  - 判定: 涨停(hit) / 未涨停(miss)
  - 记录: predicted_confidence, actual_result, hit/miss
```

### 2. 鉴别力计算

对每个子 agent:

```text
鉴别力 = avg_score(命中股票) - avg_score(未命中股票)
命中率 = 命中数 / (命中数 + 未命中数) × 100%
```

鉴别力含义:
- > 0.6: 该agent评分能有效区分涨停/非涨停
- 0.3 ~ 0.6: 有一定区分能力
- < 0.3: 区分能力弱
- < 0: 评分方向反了，需要检查

### 3. 权重调整

```text
if 鉴别力 > 0.6:    delta = +5
if 0.3 < 鉴别力 <= 0.6: delta = 0
if 0 < 鉴别力 <= 0.3:   delta = -5
if 鉴别力 <= 0:         delta = -10

新权重 = clamp(当前权重 + delta, 10, 40)
归一化: 每个权重 = 新权重 / 总和 × 100
```

### 4. 案例归档

每日保存:
- 命中最高置信度的 Top5 股票
- 误判最严重的 Top5 股票
- 供后续模型调优参考

### 5. 报告生成与推送

- 构建飞书复盘卡片
- 发送到复盘专用 webhook
- 本地存档到 reviews/ 目录

## 周期统计

| 统计维度 | 频率 | 内容 |
|---|---|---|
| 日度 | 每日15:30 | 当日准确率、鉴别力、权重调整建议 |
| 周度 | 每周日 | 本周平均准确率、权重变化趋势 |
| 月度 | 每月末 | 月度综合评估、模型调优建议 |

## 人工确认机制

权重调整建议不会自动生效:
1. 复盘报告推送到飞书
2. 人工审核调整建议
3. 确认后手动更新 weights.json 或通过 API 更新
4. 次一交易日开盘生效

## 异常处理

- 当日无预测: 发送零记录报告
- 实际结果API失败: 标记为"数据缺失"，跳过当日复盘
- 子agent连续3日鉴别力 < 0: 告警并建议检查该agent模型
