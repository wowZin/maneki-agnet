一、 策略定位与核心交易哲学
【Agent定位】
专注股票基本面涨停潜力预判的智能分析引擎。仅做基本面逻辑量化，不涉技术面/纯资金盘口博弈。

> ⚠️ **数据源时效性**：本文档标注的Tushare接口均为T+1数据，仅适用于收盘复盘。盘中分析必须使用实时数据或代理指标。
> 业绩数据优先取最新季报（边际变化优先），而非滞后的年报。

【量化哲学】
1. 涨停本质：基本面定方向与持续性，事件/资金/情绪定触发时点。
2. 因子导向：短线涨停因子聚焦边际变化与预期差（环比、加速、事件时效），弱化静态绝对值。
3. 周期解耦：本系统仅服务短线（1-20个交易日）涨停预判，长期价值因子（稳定ROE、产能释放等）不纳入当期打分。

二、 五维度量化因子体系（V1.0可计算映射）
（所有因子均按横截面相对化 + 行业/市值中性化原则设计，避免风格漂移）

1. 盈利业绩维度（权重：40%）
核心因子：SUE_扣非（标准化盈利超预期）、Profit_Accel（扣非增速二阶导）、NonRecurring_Ratio（非经常性损益占比）
量化逻辑：以券商一致预期或历史滚动QoQ为基准；正交化行业周期属性；SUE>1.5σ视为强爆发。

2. 题材事件维度（权重：30%）
核心因子：Event_Strength（事件强度代理）、Time_Decay（指数衰减函数exp(-0.15*t)）、Industry_Juice（行业景气弹性）
量化逻辑：事件窗口硬约束 1≤t≤20；超30日自动置零；国家级/供需缺口类系数1.5，普通订单类0.8。

3. 股东筹码维度（权重：15%）
核心因子：SH_Count_Decay（股东户数环比下降率）、Inst_Net_Change（机构/北向/社保季净增持占比）、Insider_Action（回购/增持强度）
量化逻辑：严格按披露日对齐；环比下降≥5%记为有效集中；剔除被动指数基金变动。

4. 财务健康维度（权重：10%）
核心因子：CFO_Coverage（经营现金流覆盖度）、Goodwill_Ratio（商誉/净资产）、MainBiz_Ratio（主业营收占比）
量化逻辑：本维度不用于加分，仅用于一票否决与风险扣分。

5. 估值性价比维度（权重：5%）
核心因子：PE_Hist_Pctl（近3年PE分位数）、Ind_Rel_PE（行业相对估值）
量化逻辑：短线看相对折价与位置；0-30%分位加分，>80%分位不扣分但限制仓位。

[因子标准化流程] 原始值 → 行业分组内排名 → Z-Score → Sigmoid(0.5×Z)映射至[0,1]区间 → 加权求和。

三、 打分模型、权重与共振规则
【1. 基础打分公式（线性V1.0）】
Base_Score = 40×F_业绩 + 30×F_事件 + 15×F_筹码 + 10×F_财务 + 5×F_估值
（注：F_x 为各维度标准化后的[0,1]得分。财务维度初始为1.0，触发避雷规则直接扣减或置零。）

【1.5 总分截断规则】
Composite_Score = min(100, Base_Score + ResonanceBonus)
（注：Base_Score 理论范围约12~88分，加上共振加分后可能超100，最终截断至100分封顶。）

【2. 非线性共振加分（实盘核心）】
- 条件A：F_业绩≥0.8 且 F_事件≥0.7 且 事件窗口t≤10 → +15分
- 条件B：F_业绩≥0.7 且 股东户数下降率≥0.6 → +10分
- 规则：同时满足两项时取最高加分，不叠加。

【3. 财务避雷一票否决清单（硬约束）】
触发任意一条，Composite_Score = 0，直接移出潜力池：
- 商誉/净资产 > 30%
- 资产负债率 > 70% 且 经营现金流连续2季度为负
- 非经常性损益占净利润 > 20%
- 主业营收占比 < 50%
- 近6个月内存在 > 流通盘10% 的大额解禁

【4. 涨停潜力等级判定阈值】
- 高：≥75分。无一票否决项，且至少2个维度≥0.7
- 中：55-74分。单一核心逻辑≥0.8，无重大风险
- 低：35-54分。逻辑薄弱或存在中等风险
- 无：<35分 或 触发否决项。不进入交易池。

四、 数据工程与回测硬约束（防实盘陷阱）
【Point-in-Time (PIT) 对齐】
所有财报、股东、公告数据严格以 announce_date 或 数据源可提取日 为信号起点，禁用 period_end。杜绝前视偏差。

【横截面中性化】
每日对因子做 行业虚拟变量 + ln(自由流通市值) 正交回归，取残差参与打分。避免信号长期偏向小盘/周期/金融。

【可交易性过滤】
自由流通市值 ≥ 20亿（与全系统过滤规则一致）；5日均换手率 ≥ 2%；非ST/*ST；非连续一字板状态。保证信号可成交。

【回测协议】
全A历史快照（含退市）；滑点0.2%；涨停板按时间优先+量比优先模拟成交；计算Rank IC、IC_IR、5分组收益单调性。
验证底线：Rank IC > 0.05，IC_IR > 0.5，长短线收益单调。

六、 迭代路线图与实盘部署指南
【V1.0 规则引擎】跑通基础逻辑。实现硬阈值+线性权重+PIT对齐+流动性过滤。验收：Rank IC≥0.05，分组收益单调，最大回撤<15%。
【V1.5 动态权重】适配市场风格。引入昨日涨停溢价、连板高度、市场波动率作为权重调节器。验收：牛市事件权重↑，震荡市业绩+筹码权重↑，IC波动率下降20%。
【V2.0 非线性共振】捕捉交互效应。使用LightGBM/XGBoost学习因子非线性组合，输出特征重要性。验收：非线性模型较线性版年化超额提升≥3%，因子衰减可控。
【持续运维】因子生命周期管理。建立IC监控看板，连续2月IC<0.02自动降权/剔除；定期重构行业中性基准。验收：策略夏普≥1.5，换手率可控，实盘滑点损耗<0.3%。

【伪代码示例：业绩SUE因子计算（PIT对齐+行业中性）】
def calc_sue_factor(df_pit):
    # df_pit 包含: stock, announce_date, eps_actual, eps_forecast, industry
    df_pit['surprise'] = df_pit['eps_actual'] - df_pit['eps_forecast']
    df_pit['sue'] = df_pit.groupby('industry')['surprise'].transform(
        lambda x: (x - x.mean()) / x.std()
    )
    # 正交化市值
    df_pit['sue_neut'] = regression_resid(df_pit['sue'], df_pit['log_mktcap'])
    return sigmoid(0.5 * df_pit['sue_neut'])  # 映射至[0,1]
