# 权重优化引擎

`scripts/optimize_ranking.py` — 基于排序质量的权重优化器。

## 原理

枚举 2000+ 种权重组合，用 [[评估指标说明]] 综合评分，找最佳权重。

**不是**看推送命中率（样本太少），而是看**全信号列表的排序质量**。

## 输出

| 输出 | 路径 |
|------|------|
| JSON 结果 | `data/weights/ranking_optimized.json` |
| 飞书推送 | 工作日 19:00 自动推送 |

## 三个核心产出版块

### 1. 信号池 vs 实战推送

```
📦 信号池(≥35): 385只(含涨停44只, 覆盖率38.9%)
🚀 实战推送(Top3): 推送3只 命中2只 命中率66.7%
```

### 2. 维度贡献率

各维度在涨停股Top3中出现的频率：

```
基本面: 88%  █████████████████
技术面: 71%  ██████████████
情绪面: 50%  █████████
短线:   51%  ██████████
资金面: 41%  ████████
```

### 3. 阈值校准曲线

不同阈值下信号池大小、覆盖率、命中率的关系。

## 当前局限

情绪面和短线博弈数据源修复后，区分度提升，优化器才能找到真正有效的权重。
